# Облачная платформа для сетей терминалов — Системы самообслуживания

Сервер сети терминалов самообслуживания – это расположенный на хостинге или в частной сети предприятия комплекс из серверного программного обеспечения и серверного оборудования.

Кастомная часть – система регистрации и управления сущностями процесса обслуживания в устройствах самообслуживания.

Система идентификации и учета узлов.

Разработана на базе ПО центра сертификации Microsoft CA.

Назначение системы — обеспечивает управление PKI инфраструктурой, целостность логической схемы сети, надежный учет и идентификацию логических узлов в условиях возможной турбулентности узлов на уровне железа (перемещения, текущие ремонты и т.п.). Предоставляет удобный и простой инструмент для администрирования узлов (добавление, изменение узла, простая процедура привязки “железа” к логической инфраструктуре). Системой осуществляется упрощенная, безопасная и автоматизированная процедура выдачи и замены ssl-сертификатов узлов, посредством эмиссии одноразовых установочных пин-кодов.

Система состоит из программной надстройка над Microsoft CA, базы данных ms sql, набора веб-сервисов и апи для узлов, программных компонентов для узлов (совместимо со всеми версиями windows), решений для встраивания административных панелей в портал управления. Также, в составе системы используется политика применения системы.

Система сбора, хранения и обработки платежей в терминалах.

Создается на базе Microsoft IIS + Microsoft SQL Server.

Система обеспечивает гарантированную доставку в центральную базу данных транзакций, сформированных на терминалах в результате операций по оплате (факты оплат через купюроприемники в терминалах, факты авторизации банками платежей по картам).

Система обеспечивает надежную очередь отправки транзакций с терминалов на сервер и отказоустойчивость к возможным проблемам скорости и качества каналов связи.

Безопасность сбора платежей обеспечивается использованием описанной выше PKI инфраструктурой, транзакционным обменом по SSL протоколу и добавлением секретной сигнатуры. Уровень безопасности транзакций адекватен уровню внешних и внутренних угроз.

Система обеспечивает сбор данных по фактам изъятия денежных кассет при инкассации терминалов.

Система состоит из масштабируемого веб-сервиса приема данных от терминалов, базы данных для хранения транзакционных пакетов, а также элементов администрирования для визуального контроля и поиска по потоку транзакций.

Ситуационный центр.

Предназначен для централизованного визуального контроля состояния сети терминалов и оборудования. Обеспечивает сбор и отображение событий и состояний сети устройств, индикацию наличия связи, индикацию проблем с приемом купюр, печатью квитанций, а также контроль текущих версий ПО.

Состоит из масштабируемого веб-сервиса для приема пакетов контроля и регистрации событий в сети, базы данных для хранения потока событий, а также интерфейса администрирования.

Система дистанционного обновления ПО терминалов.

Назначение системы — надежная доставка программных пакетов обновления версий ПО на терминалы. Обеспечивает доставку наборов файлов, внутренний контроль размеров и целостности пакетов. Доставка осуществляется через FTP-сервер. При этом, существует возможность поэтапного распределения новых версий для обеспечения тестовых и опытных периодов эксплуатации на ограниченном сегменте сети и недопущения попадания ошибочных версий на всю сеть. Система обеспечивает автоматизацию процессов идентификации версий, а также подготовки пакетов файлов. Система обновления является абсолютно отказоустойчивой к проблемам скорости и качества каналов связи.

Задачи, в которых применяются наши серверные решения:

Предоставление личного кабинета владельцам комплексов Электронный кассир;

Встраивание возможностей и компонентов платформы в заказные разработки сетей самообслуживания.
