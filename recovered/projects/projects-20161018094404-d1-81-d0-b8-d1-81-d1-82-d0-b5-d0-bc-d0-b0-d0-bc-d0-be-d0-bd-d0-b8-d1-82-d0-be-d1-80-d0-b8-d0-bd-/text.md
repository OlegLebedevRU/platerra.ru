# Система управления для билетных терминалов Аэроэкспресс — Software for Self-Service

Качество услуг в сети билетных автоматов Аэроэкспресс обеспечено во многом за счет разработанного нашей компанией программного комплекса мониторинга и управления для этой распределенной сети. В рамках проекта реализованы задачи централизованного мониторинга, прием платежных транзакций от автоматов, диагностики билетных автоматов и обеспечения рабочих мест диспетчера по обслуживанию оборудования и решению проблем со сбоями в приеме оплаты или выдаче сдачи. Также, разработаны элементы обратной связи, подсистема формирования отчетов и многое другое.

Сервер управления с точки зрения пользователей системы выполняет функции мониторинга и аналитики, логически подключается к сети билетных автоматов, а так же к офисной сети, где находятся всевозможные АРМ:

1) АРМ Претензионной работы – рабочее место, на котором происходит разбор претензий пользователей (не распечатанный билет, не выданная сдача и так далее).

2) АРМ Администратора – рабочее место, в котором ведутся настройки системы, запускается подпрограмма по обновлению ПО на билетных автоматах, создаются карты доступа.

3) АРМ Аналитики – рабочее место, в котором ведутся отчеты по блиц-опросам, редактируются билеты блиц-опросов, ведется статистика по посещению страниц билетных автоматов.

4) АРМ Технического мониторинга – рабочее место технического мониторинга, в котором визуально отражается техническая карта сети автоматов с отражением состояния устройств, а так же присутствует функционал удаленного управления оборудованием. Так же на данном рабочем месте выводятся все оперативный сообщения о нештатных ситуациях. Данный АРМ предназначен для круглосуточного наблюдения за работоспособностью всей сети.

Функции сервера мониторинга.

1) Приём и хранение информации по техническому состоянию устройств.

2) Приём и хранение экстренных сообщений от устройств сети. Оперативная передача сообщений на АРМ мониторинга.

3) Приём и хранение информации по всем сессиям продажи билетов, включая информацию по не выданной сдаче и нераспечатанным билетам.

4) Приём и хранение информации по внесениям, выплатам, открытию и закрытию смен.

5) Обновление программного обеспечения билетных автоматов. Рассылка дистрибутива ПО и контроль обновления.

6) Сбор и хранение статистики посещения HTML страниц интерфейса.

7) Сбор и хранение информации по проведённым блиц-опросам.

8) Формирование ежедневных технических отчетов и отправка их по E-mail в формате Microsoft Excel.

Функции АРМ претензионной работы.

Назначение АРМ претензионной работы по билетам: Разбор ситуации с проблемными продажами, такими как не выданная сдача и не распечатанный билет. После решения проблемы с билетом в базе сохраняется отметка о дате и варианте решения проблемы.

По билету с не выданной сдачей может быть либо обработан путем перевода денежных средств на мобильный телефон через платежную систему КИБИРПЛАТ, либо путем перевода денег на банковский счет клиента.

Не распечатанный билет может быть отправлен на другой терминал для распечатки билета не в фискальном режиме.

Дополнительной функцией АРМ претензионной работы является окно бегущих транзакций. Отображается в режиме ожидания мониторинга. В нем виде бегущего списка отображаются платежные транзакции, которые поступают на сервер. Проблемные транзакции выкрашиваются красным цветом, Покупка билетов стандартного класса желтым цветом, бизнес класса зелёным цветом. Данное окно позволяет оценить динамику продаж.
