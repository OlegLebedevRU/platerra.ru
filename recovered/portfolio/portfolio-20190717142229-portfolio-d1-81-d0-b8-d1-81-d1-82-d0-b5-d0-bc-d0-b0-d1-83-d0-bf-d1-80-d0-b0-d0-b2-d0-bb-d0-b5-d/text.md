Система идентификации и учета узлов.

Создается на базе ПО центра сертификации Microsoft CA.

Назначение системы — обеспечивает управление PKI инфраструктурой, целостность логической схемы сети, надежный учет и идентификацию логических узлов в условиях возможной турбулентности узлов на уровне железа (перемещения, текущие ремонты и т.п.). Предоставляет удобный и простой инструмент для администрирования узлов (добавление, изменение узла, простая процедура привязки “железа” к логической инфраструктуре). Системой осуществляется упрощенная, безопасная и автоматизированная процедура выдачи и замены ssl-сертификатов узлов, посредством эмиссии одноразовых установочных пин-кодов.

Система состоит из программной надстройка над Microsoft CA, базы данных ms sql, набора веб-сервисов и апи для узлов, программных компонентов для узлов (совместимо со всеми версиями windows), решений для встраивания административных панелей в портал управления. Также, в составе системы используется политика применения системы.

Система сбора, хранения и обработки платежей в постаматах.

Создается на базе Microsoft IIS + Microsoft SQL Server.

Система обеспечивает гарантированную доставку в центральную базу данных транзакций, сформированных на постаматах в результате операций по оплате (факты оплат через купюроприемники в постаматах, факты авторизации банками платежей по картам).

Система обеспечивает надежную очередь отправки транзакций с постаматов на сервер и отказоустойчивость к возможным проблемам скорости и качества каналов связи.

Безопасность сбора платежей обеспечивается использованием описанной выше PKI инфраструктурой, транзакционным обменом по SSL протоколу и добавлением секретной сигнатуры. Уровень безопасности транзакций адекватен уровню внешних и внутренних угроз.

Система обеспечивает сбор данных по фактам изъятия денежных кассет при инкассации постаматов.

Система состоит из масштабируемого веб-сервиса приема данных от постаматов, базы данных для хранения транзакционных пакетов, а также элементов администрирования для визуального контроля и поиска по потоку транзакций.

Ситуационный центр.

Предназначен для централизованного визуального контроля состояния сети постаматов и оборудования. Обеспечивает сбор и отображение событий и состояний сети устройств, индикацию наличия связи, индикацию проблем с приемом купюр, печатью квитанций, а также контроль текущих версий ПО.

Состоит из масштабируемого веб-сервиса для приема пакетов контроля и регистрации событий в сети, базы данных для хранения потока событий, а также интерфейса администрирования.

Система дистанционного обновления ПО постаматов.

Назначение системы — надежная доставка программных пакетов обновления версий ПО на постаматы. Обеспечивает доставку наборов файлов, внутренний контроль размеров и целостности пакетов. Доставка осуществлятся через FTP-сервер. При этом, существует возможность поэтапного распределения новых версий для обеспечения тестовых и опытных периодов эксплуатации на ограниченном сегменте сети и недопущения попадания ошибочных версий на всю сеть. Система обеспечивает автоматизацию процессов идентификации версий, а также подготовки пакетов файлов. Система обновления является абсолютно отказоустойчивой к проблемам скорости и качества каналов связи.

Система регистрации и управления сущностями процесса доставки посылок через постаматы (основные сущности — посылки, получатели, роли, узлы).

Система предназначена для управления жизненным циклом посылок, доставляемых посредством сети постаматов.

Система состоит из внутреннего АПИ для постаматов, центральной базы данных с сущностями, событиями и состояниями, служб для интеграции с внешними сервисами (гейты СМС и емайл-оповещений, гейты обмена данными с основной системой заказчика и т.п.), а также портала для обеспечения ролевых функций персонала сети (администраторы, операторы регистрации посылок и отправлений, курьеры).
